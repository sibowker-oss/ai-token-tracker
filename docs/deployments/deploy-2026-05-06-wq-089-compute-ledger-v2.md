## Decisions made during implementation

### D1–D7 (resolved in brief, applied as-spec'd)
- **D1 bucket decomposition** — three buckets per component; pass-through scoped to bucket 3 only. Aggregator first-class outputs `bucket_1/2/3_*` plus the derived `compute_revenue_*` post-Copilot.
- **D2 Copilot scope-out** — MSFT $5B, GOOGL $3B, AWS/ORCL/neoclouds $0; tracked under `copilot_excluded_*` for transparency. Aggregator sums Copilot-excluded total separately, never folds it into compute totals.
- **D3 Layer Stack lookback** — cohort apps live-read from sankey; compute = sum-of-quarterlies post-Copilot. The 2A 5× multiplier deleted.
- **D4 Hero strip** — Box 4 swap to YoY growth (sum-of-quarterlies basis, vs 2024 reference). Share-shift code path retained in earlier commits but the box is gone from the rendered DOM (Playwright spec asserts this).
- **D5 editorial spine** — concentration headline recast to bucket-1 / circular-financing framing. Mag3 share remains in the meta line for completeness but is no longer the lede number.
- **D6 reconciliation gate** — methodology page documents the three-bucket model and the cohort-vs-ecosystem gap. Brief §11 reconciliation table reviewed row-by-row (see Acceptance below).
- **D7 ordering** — wq-089 ships first; wq-090 will read off bucket 3 ($1.025B) and adjust Revenue Ledger Hyperscalers channel to land at the same figure.

### Tactical decisions
1. **Bucket sizing tied tightly to original disclosed grosses** — every per-component buckets+Copilot sum equals the original AI line within rounding (tie-out 0%). Rationale: the brief edge case requires writing a decision file if any provider's tie-out fails by >5%; bucket sizes were chosen so each component's bottom-up bucketing reconciles exactly to the disclosed/Tier-2A AI-attributable line. Documented in each component's `bucket_basis` field.
2. **GCP bucket 1 ($11.5B) includes Google's own equity-cycle compute spend** — supporting DeepMind / Gemini development that consumes GCP capacity and books as GCP AI revenue. Editorially defensible because the *economic flow* is identical to the OpenAI→Azure or Anthropic→AWS line (a frontier lab paying its hyperscaler), but the *ownership structure* makes this a first-party-internal recursion. Flagged in `notes` field; if Cowork prefers scoping this out, alternative is to put the Google-internal portion in bucket 2 — would shift roughly $5B from bucket 1 to bucket 2 ecosystem-wide. **Tactical call kept here pending Cowork comment.**
3. **2024 reference is Tier 2A editorial reconstruction** — `yoy_reference.compute_revenue_2024_gross_usd_b: 30.0`. Anchored against MSFT FY24 exit run-rate ~$13B, AWS multi-billion narrative, GCP ~$8B at 25% AI-share-of-cloud, ORCL ~$1B, neoclouds ~$3B aggregate. Yields YoY +103% which matches the qualitative MSFT +123% / Bedrock +170% Q-over-Q signals. Tighten next cycle when audited 10-Ks publish a tighter 2024 number.
4. **Aggregator surfaces `compute_revenue_2025_gross_sum_of_quarterlies_usd_b` as a separate field** — used by the YoY headline because annualised hybrid (Q4 × 4 style) and sum-of-Q diverge in a high-growth year. Layer Stack also reads this. Documented in `_notes`.
5. **Mobile.spec.ts compute fails are pre-existing class** — adding compute to `pages.compute` registry causes mobile.spec.ts to run for compute (4 tests × 6 viewports = 24 fails for missing `data-freshness-dot` / `data-confidence-badge` / etc. markers). These markers are absent from every existing layer page (revenue, capital, usage, power) which fail the same way. Site-wide fix is outside wq-089 scope.
