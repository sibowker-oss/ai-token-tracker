2. ARR → Collected Revenue Conversion

Method: ARR is adjusted for reporting methodology (OpenAI uses "best 4-week x 12" = 0.88 haircut), then converted to collected revenue using growth-adjusted calculation (log-mean for hyper-growth, simple average for moderate). Finally, buyer segment collection factors are applied based on who is paying — not the channel they use.

Buyer segments (not channels):
• Consumer — individual subscribers (ChatGPT Plus, Claude Pro, Gemini). Collection: 0.95 (monthly billing, some churn)
• SME — small/medium businesses, AI-native apps, startups buying via API. Collection: 0.90 (moderate churn)
• Enterprise — large enterprise contracts (direct, via SaaS, or via hyperscaler). Collection: 0.80 (long sales cycles, ramp periods)

Note: API is a channel, not a buyer segment. Most API revenue comes from SMEs (AI-native apps and startups). Enterprise API usage is growing but still minority.
